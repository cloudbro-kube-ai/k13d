# Web UI Settings > AI 탭 직관적 재구성 계획

## 목표
Web UI의 Settings > AI 탭에서 프로파일 관련 UI를 제거하고, 저장 시 자동 연결 테스트를 추가하여 TUI와 동일한 사용자 경험 제공

## 변경 대상 파일

### 1. `pkg/web/static/index.html`
- **제거할 섹션**: "Saved Model Profiles" (Lines 1108-1172)
  - `<div id="models-list">` 
  - "Add Model Profile" 버튼
  - `<div id="add-model-form">` 전체 폼
  
- **변경할 버튼**: "Test Connection" 버튼 제거 (저장 시 자동 테스트)
  - Lines 1033-1036: Test Connection 버튼 제거

- **변경할 버튼**: "Save LLM Configuration" 버튼에서 저장 후 자동 연결 테스트 추가

### 2. `pkg/web/static/js/features/settings.js`
- **제거할 함수**:
  - `showAddModelForm()`
  - `hideAddModelForm()`
  - `addModelProfile()`
  - `saveModelProfile()`
  - `updateNewModelToolSupportWarning()`
  - `renderModelsList()`
  
- **변경할 함수**:
  - `saveLLMConfig()` - 저장 후 자동으로 `testLLMConnection()` 호출
  - `testLLMConnection()` - 버튼 제거에 따라 자동 호출되도록 변경

### 3. `pkg/web/handlers_settings.go`
- 이미 수정 완료됨 (단일 모델 관리로 변경)

## 검증 방법

1. **빌드 확인**: `go build ./...`
2. **Web UI 테스트**:
   - Settings > AI 탭 접속
   - Provider 선택 → Model 입력 → Endpoint/APIKey 입력
   - Save 클릭 시 자동으로 연결 테스트 수행 확인
   - "Saved Model Profiles" 섹션이 없는지 확인
3. **TUI 테스트**: `./k13d` 실행 후 Shift+O로 Settings 확인
